# Модуль greating-VM

## Описание
Модуль greating-VM предназначен для создания виртуальных машин в Yandex Cloud. 
Этот модуль не получает подсети самостоятельно — подсеть передается из корневой конфигурации.

## Функционал
- Создает виртуальную машину в указанной подсети
- Настраивает SSH доступ
- Предоставляет параметры для конфигурации ВМ
- Опционально включает NAT

## Входные переменные

### Обязательные
- `subnet_id` - ID подсети (передается из корневой конфигурации)
- `zone` - Зона доступности 
- `vm_name` - Имя виртуальной машины

### Опциональные
- `platform_id`  -платформа ЦП
- `cores` - Количество vCPU
- `memory` - Объем RAM в ГБ
- `core_fraction` - Производительность (5, 25, 50, 100)
- `image_family` - Семейство образа ОС
- `disk_size` - Размер диска в ГБ
- `disk_type` - Тип диска
- `nat`  - Включить NAT
- `ssh_keys`  - SSH ключи
- `ssh_user` - Пользователь SSH

## Выходные данные
- `instance_id` - ID ВМ
- `instance_name` - Имя ВМ
- `internal_ip` - Внутренний IP
- `external_ip` - Внешний IP (если nat=true)
- `zone` - Зона ВМ
- `subnet_id` - ID подсети
- `status` - Статус ВМ

## Архитектура использования
Этот модуль используется корневой конфигурацией, которая:
1. Вызывает модуль `network-data` для получения подсетей
2. Выбирает подсеть по указанной зоне
3. Передает `subnet_id` этому модулю
